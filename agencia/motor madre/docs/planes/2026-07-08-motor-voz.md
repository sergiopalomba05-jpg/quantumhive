## Resumen

Motor de voz propio multi-voz con clonación zero-shot. Cada avatar/cliente tiene su voz registrada con 5-15 segundos de audio de referencia. El motor genera audio en streaming (el avatar empieza a hablar antes de terminar de procesar).

## Stack Técnico

| Componente | Tecnología | Notas |
|------------|-----------|-------|
| Modelo TTS | **F5-TTS** | Open-source, streaming, clonación zero-shot |
| Alternativa | XTTS v2 | Si F5-TTS no convence |
| Servidor | **FastAPI** + Python | StreamingResponse para audio en tiempo real |
| GPU | NVIDIA L4 (24GB VRAM) | VM comfyui-l4 en GCP |
| Almacenamiento audios | **Supabase Storage** | Audios de referencia 5-15 seg por voz |
| Caché | Local + Supase | Audios pre-renderizados para greetings |
| Aceleración | DeepSpeed / TensorRT | RTF < 0.2 (10 seg de audio en < 2 seg) |

## Arquitectura

```
PWA (Cloud Run)
    │
    ▼
POST /v1/voice/generate { text, voice_id, use_cache }
    │
    ▼
FastAPI Server (VM GPU)
    │
    ├── Buscar audio de referencia → voices/{voice_id}_ref.wav
    │
    ├── ¿use_cache=true?
    │   ├── SÍ → Buscar en cache/ → si existe, devolver
    │   └── NO → Generar completo, guardar en cache/
    │
    └── StreamingResponse (bytes PCM/WAV)
         │
         ▼
    F5-TTS.infer_stream() → chunks → cliente recibe audio
```

## Endpoints API

### POST /v1/voice/generate
Genera audio en streaming con clonación de voz.

**Request:**
```json
{
  "text": "¡Hola! Bienvenido a La Escaloneta, ¿qué vas a pedir?",
  "voice_id": "mesera-porteña",
  "use_cache": false
}
```

**Response:** Stream de audio WAV/PCM

### POST /v1/voice/clone
Registra una nueva voz con audio de referencia.

**Request:**
```json
{
  "voice_id": "nuevo-avatar",
  "audio_b64": "<wav/mp3 base64>",
  "ref_text": "Texto opcional del audio de referencia"
}
```

### GET /v1/voice/health
Health check del motor.

## Código Base (FastAPI)

```python
import os
from fastapi import FastAPI, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import torch
from f5_tts.api import F5TTS

app = FastAPI(title="Motor de Voz")

device = "cuda" if torch.cuda.is_available() else "cpu"
tts_model = F5TTS(model_type="F5-TTS", device=device)

class VoiceRequest(BaseModel):
    text: str
    voice_id: str
    use_cache: bool = False

def generate_audio_stream(text: str, ref_audio_path: str):
    chunks = tts_model.infer_stream(
        gen_text=text,
        ref_audio=ref_audio_path,
        ref_text="",
    )
    for chunk in chunks:
        yield chunk

@app.post("/v1/voice/generate")
async def generate_voice(request: VoiceRequest):
    ref_audio_path = f"voices/{request.voice_id}_ref.wav"
    
    if not os.path.exists(ref_audio_path):
        raise HTTPException(status_code=404, detail="Voz no encontrada")

    if request.use_cache:
        cache_path = f"cache/{request.voice_id}_{hash(request.text)}.wav"
        if os.path.exists(cache_path):
            return StreamingResponse(open(cache_path, "rb"), media_type="audio/wav")
        tts_model.infer_to_file(
            gen_text=request.text,
            ref_audio=ref_audio_path,
            output_path=cache_path
        )
        return StreamingResponse(open(cache_path, "rb"), media_type="audio/wav")

    return StreamingResponse(
        generate_audio_stream(request.text, ref_audio_path),
        media_type="audio/wav"
    )

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

## Casos de Uso

### 1. Greeting pre-grabado (splash screen)
- `use_cache: true`
- Se genera una vez, se guarda en Supabase Storage
- Costo: $0 por reproducción

### 2. Respuestas dinámicas del chat
- `use_cache: false`
- Streaming puro, el avatar habla mientras genera
- Latencia: 100-200ms primer chunk

### 3. Voice Cloning SaaS (futuro)
- Clientes suben 5 seg de audio
- Se registra como nueva voz
- Se cobra por caracteres generados

## Integración con Supabase

1. **Storage:** Audios de referencia en bucket `voices/`
2. **DB:** Tabla `voices` con id, voice_id, ref_audio_url, created_at
3. **Credits:** Tabla `voice_credits` para cobrar por uso SaaS
